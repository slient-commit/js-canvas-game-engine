JCGE Editor

A visual game editor for the JS Canvas Game Engine. Build 2D games with scenes, layers, sprites, physics, UI, audio, and export standalone HTML5 games.

Electron + React HTML5 Canvas JavaScript

Getting Started

Creating a New Project

  1. Click New in the toolbar
  2. Enter a project name, canvas width, and canvas height
  3. Choose a folder for the project
  4. The editor creates the project folder with the engine, asset directories, and a .jcge project file

Open & Save

  • Open — Opens a .jcge project file
  • Save — Saves the current project. First save prompts for a location; subsequent saves overwrite
Tip
The editor shows "Unsaved changes" in the toolbar when there are pending changes.

Project Structure

MyGame/ ├── MyGame.jcge # Project file (JSON) ├── engine/ # Engine source (auto-copied) │ ├── engine-core.js │ └── game-loader.js ├── assets/ │ ├── sprites/ # Imported images (PNG, JPG, etc.) │ └── audio/ # Imported audio (MP3, WAV, OGG) └── scripts/ # Custom JS script files

Scene Hierarchy (Left Panel)

A tree view of all scenes, layers, and objects:

  • Scenes — click to select, shows default badge on the default scene
  • Layers — nested under scenes, shows (UI) badge for UI layers
  • Game Objects — listed under their layer
  • Elements — listed under their layer

Use the + Add Scene, + Layer, + UI Layer, and + Add Object buttons to create new items.

Center Panel (Preview / Code / Files)

The center panel has three tabs:

  • Preview — Live game canvas with pixel rulers
  • Code — Full-size scene script editor with helpers and snippets
  • Files — External script file editor for shared JavaScript files

The Code tab auto-activates when a script error occurs, showing the error message.

Properties Panel (Right Panel)

Context-sensitive inspector that changes based on selection:

  • Scene selected — name, default checkbox, camera settings
  • Layer selected — name, z-order, UI layer toggle, delete button
  • Object selected — name, position, velocity, opacity, visibility, sprite, delete button

Asset Manager (Bottom Panel)

Manages sprite and audio assets for the project.

  • Import button to add images (.png .jpg .gif .bmp .webp) and audio (.mp3 .wav .ogg)
  • Sprite thumbnails displayed in a grid
  • Audio files can be previewed with click-to-play

Scenes

Scenes are the top-level containers for game content. Each scene has its own layers, camera, and script hooks. Only one scene is active at a time.

PropertyDescription
nameDisplay name of the scene
isDefaultIf true, this scene loads first when the game starts
cameraOptional camera (fixed or world)
layersArray of layers containing game objects and elements
scriptThree hooks: onCreate, onUpdate, onDestroy

Scene Scripts

Each scene has three script hooks you can edit in the Code tab:

HookWhen it runsAvailable variables
onCreateOnce, when the scene startsengine, this
onUpdateEvery frameengine, this, elapsedTime
onDestroyOnce, when the scene endsengine, this

Inside scripts you can use:

// Find objects by name
var player = this.findByName('Player');

// Read input
if (engine.input.isKeyDown(Keys.ArrowRight)) {
  player.position.X += 200 * elapsedTime;
}

// Switch scenes
engine.goToScene(engine.scenes[1], { score: 100 }, 'fade', 0.5);

Scene Transitions

Use engine.goToScene(scene, data, transition, duration) to switch scenes.

TransitionEffect
'none'Instant switch
'fade'Fade to black, then fade in
'slide-left'Slide left transition
'slide-right'Slide right transition

The data parameter can be retrieved in the new scene with this.getIncomingData().

Layers

Layers control rendering order and grouping. Objects in higher z-order layers render on top.

TypeDescription
LayerStandard layer, affected by camera position. Use for game world content.
UILayerScreen-space layer, ignores camera. Use for HUD, menus, and overlays.

Every scene starts with a default ground layer. Add more layers via the hierarchy panel.

Game Objects

GameObjects are the primary interactive entities in a scene. They support sprites, movement, and collision detection.

PropertyTypeDescription
nameStringIdentifier for findByName()
positionVec2World position (X, Y)
velocityVec2Movement per second (applied automatically unless static)
opacityNumber0 (invisible) to 1 (opaque)
showItBooleanVisibility toggle
isStaticBooleanIf true, velocity is not applied
spriteSpriteVisual appearance
// Create a game object in script
var spr = new Sprite(32, 32, 'assets/sprites/player.png');
var player = new GameObject(spr, new Vec2(100, 100));
player.name = 'Player';
layer.registerGameObject(player);

Elements

Elements are visual objects for static decorations or UI components. The editor supports four types:

UILabel

Text-rendering UI element.

var label = new UILabel('Score: 0', new Vec2(10, 10), {
  fontSize: 20,
  fontFamily: 'monospace',
  color: 'white'
});
uiLayer.registerElement(label);

// Update text later
label.setText('Score: 500');
OptionDefaultDescription
fontSize14Font size in pixels
fontFamily'monospace'CSS font family
fontStyle'normal'CSS font style (normal, bold, italic)
color'white'Text color

UIButton

Interactive button element with hover and press states.

var btn = new UIButton(new Vec2(100, 200), new Size(120, 40), {
  label: 'Start',
  normalColor: '#333',
  hoverColor: '#555',
  pressedColor: '#777',
  borderColor: '#888'
});
btn.onClick = function() { /* handle click */ };
uiLayer.registerElement(btn);

UIPanel

A rectangular panel for backgrounds and containers.

var panel = new UIPanel(new Vec2(50, 50), new Size(200, 150), {
  fillColor: 'rgba(0,0,0,0.5)',
  borderColor: '#888',
  borderWidth: 2
});
uiLayer.registerElement(panel);

UICircle

A circle shape element for drawing filled or outlined circles.

var circle = new UICircle(new Vec2(100, 100), {
  radius: 30,
  fillColor: '#3498db',
  borderColor: '#2980b9',
  borderWidth: 2
});
layer.registerElement(circle);

Rectangle (Shape)

In the editor, use the Rectangle object type to place colored rectangles on any layer. These are exported as UIPanel instances at build time.

Circle (Shape)

In the editor, use the Circle object type to place circles on any layer. These are exported as UICircle instances at build time.

Visual Editing: All UI elements (Label, Button, Panel) and shape primitives (Rectangle, Circle) render visually on the editor canvas in edit mode. Select them to see their bounding box, drag to reposition, and edit their properties in the Properties panel.

Parent-Child Attachments

Attach any object (GameObject or Element) to a parent so it automatically follows the parent's position with an X/Y offset. This is useful for turrets on vehicles, HP bars above characters, weapon attachments, etc.

Using in the Editor

  1. Create the parent object (e.g. tank body) and one or more child objects (e.g. turret, HP bar)
  2. Select the child object
  3. In the Properties Panel, find the Attachment section
  4. Choose the parent from the Parent dropdown — the offset is auto-calculated from current positions
  5. Adjust Offset X / Offset Y manually if needed
Visual feedback: A yellow dashed line connects child to parent in edit mode. Child objects show a ↵ indicator in the Scene Hierarchy. Dragging a child updates its offset relative to the parent.

Using in Code

// Turret follows tank body
turret.attachTo(tankBody, 10, -15);

// HP bar follows player
hpBar.attachTo(player, 0, -20);

// Detach later
turret.detach();

The engine auto-updates child positions each frame before rendering. Works across layers — a UI element can follow a game object in a different layer.

Cameras

Cameras control the viewport into the game world. Add a camera from the Properties panel when a scene is selected.

Fixed Camera

A static viewpoint. The camera does not follow objects.

var cam = new Camera(960, 540, 960, 540, 10);
// (screenW, screenH, levelW, levelH, speed)
this.setCamera(cam);

World Camera

Follows a game object with smooth interpolation, zoom, and shake support.

var cam = new WorldCamera(960, 540, 3000, 2000, 10);
cam.smoothFollow = true;
cam.followLerp = 0.1;
this.setCamera(cam);

// In onUpdate: follow the player
cam.setPositionTo(player, elapsedTime);

// Zoom and shake
cam.setZoom(1.5);
cam.shake(5, 0.3); // intensity, duration
PropertyDefaultDescription
smoothFollowtrueEnable smooth interpolation when following
followLerp0.1Interpolation factor (0 = no movement, 1 = instant)
_zoom1Zoom level (1 = normal, 2 = 2x zoom in)

Sprite & SpriteSheet

Sprite

var spr = new Sprite(width, height, imagePath);
changeImage(path)
Change the sprite's image at runtime.

SpriteSheet

var sheet = new SpriteSheet(
  'walk',    // name
  64, 64,   // frame width, height
  6,         // frame speed (frames per update)
  0, 7,      // start frame, end frame
  'assets/sprites/character.png',
  false,     // once (false = loop)
  null,      // callback on end
  -1         // once_max_frame
);

SpriteAtlas

A sprite atlas holds named regions within a single image. Unlike a SpriteSheet (uniform grid), each region can have its own position and size. Ideal for images where sprites are packed at arbitrary locations (e.g., a car with 8 directions, each a different size and position in the image).

var atlas = new SpriteAtlas('assets/sprites/car.png', {
  'north':     { x: 0,   y: 0,   width: 48, height: 64 },
  'northeast': { x: 60,  y: 10,  width: 56, height: 60 },
  'east':      { x: 130, y: 5,   width: 64, height: 48 },
  'southeast': { x: 200, y: 12,  width: 56, height: 60 },
  'south':     { x: 270, y: 0,   width: 48, height: 64 },
  'southwest': { x: 340, y: 12,  width: 56, height: 60 },
  'west':      { x: 410, y: 5,   width: 64, height: 48 },
  'northwest': { x: 480, y: 10,  width: 56, height: 60 }
}, 'north');  // default region

// Use with a GameObject
var car = new GameObject(atlas, new Vec2(100, 100));
layer.registerGameObject(car);

// Switch to a different region
atlas.setRegion('east');  // width & height update automatically

Constructor

new SpriteAtlas(imagePath, regions, defaultRegion)
ParameterTypeDescription
imagePathStringPath to the atlas image file
regionsObjectMap of region names to { x, y, width, height }
defaultRegionStringName of the initial region to display

Methods

setRegion(name)
Switch to a named region. Updates width and height to match the region.
getRegion()
Returns the current region's { x, y, width, height } in the atlas image.
getRegionNames()
Returns an array of all region names.

Editor Support

In the Properties panel, select SpriteAtlas from the sprite Type dropdown. Then:

  1. Choose an image asset
  2. Click + under Regions to add named regions
  3. Set X, Y, W, H (source rectangle in the image) for each region
  4. Select the active region from the Region dropdown

In scene scripts, switch regions at runtime with atlas.setRegion('name').

When to use which
  • Sprite — Single image, one frame
  • SpriteSheet — Single image, uniform grid of frames (animation)
  • SpriteAtlas — Single image, named regions at arbitrary positions and sizes

Chroma Key

Remove a solid background color from any sprite type. The engine processes the image once on load, making matching pixels transparent. Works with Sprite, SpriteSheet, and SpriteAtlas.

// Remove green background
var spr = new Sprite(64, 64, 'player.png');
spr.setChromaKey('#00FF00');

// Custom tolerance (0–255, default 30)
spr.setChromaKey('#00FF00', 50);

// Works on SpriteSheet and SpriteAtlas too
sheet.setChromaKey('#FF00FF', 20);
atlas.setChromaKey('#0000FF');
ParamTypeDescription
colorstringHex color to remove, e.g. '#FF00FF'
tolerancenumberMatching tolerance (0–255). Higher = more aggressive. Default: 30

Editor: In the Properties panel, set the Chroma Key color picker and adjust Tolerance. Click × to clear.

Chroma key is applied once at load time (no per-frame cost). The original image is preserved internally — calling setChromaKey again re-processes from the original.

Exporting Your Game

  1. Click Export in the toolbar
  2. Choose an output folder
  3. The editor generates a complete standalone game:
ExportedGame/ ├── index.html # Standalone HTML page ├── game.js # Generated scene classes & bootstrap ├── _sounds.js # Base64 audio cache (if audio assets exist) ├── engine/ # Engine files ├── assets/ # Sprite & audio files └── scripts/ # Custom JS files

The exported game uses a script loader that loads files in order: audio cache, custom scripts, then game.js. The engine intro scene plays before your default scene.

Note
The project must be saved before exporting. Audio files are embedded as base64 in _sounds.js to avoid CORS issues when opening index.html locally.

Code Snippets

Available in the Code tab's helper panel. Click to insert at cursor.

Arrow Keys Movement

var speed = 200;
var obj = this.findByName('OBJECT_NAME');
if (engine.input.isKeyDown(Keys.ArrowLeft))  obj.position.X -= speed * elapsedTime;
if (engine.input.isKeyDown(Keys.ArrowRight)) obj.position.X += speed * elapsedTime;
if (engine.input.isKeyDown(Keys.ArrowUp))    obj.position.Y -= speed * elapsedTime;
if (engine.input.isKeyDown(Keys.ArrowDown))  obj.position.Y += speed * elapsedTime;

Collision Check

var a = this.findByName('OBJECT_A');
var b = this.findByName('OBJECT_B');
if (a.collisionWith(b)) {
  // handle collision
}

Boundary Clamp

var obj = this.findByName('OBJECT_NAME');
var w = engine.canvas.width;
var h = engine.canvas.height;
obj.position.X = Math.max(0, Math.min(w - obj.sprite.width, obj.position.X));
obj.position.Y = Math.max(0, Math.min(h - obj.sprite.height, obj.position.Y));

Play Sound Effect

var sfx = new Sound('assets/audio/FILENAME.mp3', 80);
sfx.play();

Play Background Music

if (!this.bgMusic) {
  this.bgMusic = new Sound('assets/audio/FILENAME.mp3', 50, true);
  engine.sound.playMusic(this.bgMusic);
}
